# 通用行程頁面規範（可套用任何行程）

## 1. 目的
- 本規範用於建立「可重複使用」的行程頁面。
- 適用於任何城市、天數、主題，不綁定特定目的地。

## 2. 資料結構（每一筆行程）
- 建議欄位：
1. `time`：時間（如 `10:00`、`下午`、`餐後`）
2. `title`：標題
3. `desc`：簡述
4. `type`：分類（`transport` / `sight` / `food` / `hotel`）
5. `address`：地址（中英可並列）
6. `phone`：電話（可空）
7. `amount`：費用（可空）
8. `image`：圖片 URL（可空）
9. `realImage`：是否為真實照片（`true/false`）
10. `iconFixed`：強制 icon（可空）
11. `colorFixed`：強制顏色（可空）
12. `iconExtra`、`colorExtra`：第二個 icon（可空）

## 3. Icon 規則（預設自動判斷）
1. 交通：`fa-car`
2. 飛機/航班：`fa-plane`
3. 住宿/入住/退房：`fa-hotel`
4. 餐飲：`fa-utensils`
5. 其他景點：`fa-location-dot`

## 4. 顏色規則（對齊上方分類）
1. 交通 `Transport`：`text-sky-500`
2. 景點 `Attraction`：`text-emerald-500`
3. 餐飲 `Food`：`text-yellow-500`
4. 住宿 `Hotel`：`text-indigo-500`

## 5. 覆寫優先級
1. 若有 `iconFixed`，優先使用，不走自動判斷。
2. 若有 `colorFixed`，優先使用，不走自動配色。
3. 若有 `iconExtra`，在主 icon 旁顯示第二個 icon。

## 6. 展開資訊框規則
- 每筆行程可點擊展開。
- 展開欄位顯示邏輯：
1. 地址：永遠顯示（若有值）
2. 電話：僅在看起來像電話號碼時顯示（例：含 `+` 與數字段）
3. 金額：`amount` 有值才顯示
4. 圖片：僅 `realImage: true` 且 `image` 有值時顯示

## 7. 內容品質規範
1. 地名與飯店建議中英並列（提升辨識與可搜尋性）
2. 費用盡量標註幣別（如 `NTD`、`IDR`、`USD`）
3. 時間軸順序要一致（避免同日倒序）
4. 行程描述保持短句，細節放展開區

## 8. 樣式一致性規範
1. 單 icon 行程不可因雙 icon 行程而位移
2. 雙 icon 僅影響該筆，不能影響全局排版
3. 分類顏色只用四類（交通/景點/餐飲/住宿）
4. 未定義類型一律落到景點（綠色）或使用 `iconFixed`

## 9. 維護流程（新增行程時）
1. 先填基本欄位：`time`、`title`、`desc`、`type`
2. 再補細節欄位：`address`、`amount`、`phone`
3. 最後補圖片：有真實來源才設 `realImage: true`
4. 若 icon 不符合語意，用 `iconFixed` / `colorFixed` 單點修正

## 10. 禁忌
1. 不要把未知電話硬顯示成 `待補`
2. 不要放不明來源或非地點照片作為「真實照片」
3. 不要為單一項目改壞全局規則（應用覆寫欄位）
